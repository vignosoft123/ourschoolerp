8. Common Errors & Fixes

Error MessageCauseFix
FTP login failed — update 'pass'... cPanel/FTP password was changed Update "pass" in FTP_CONFIGS in python/.env, restart Python server
FTP directory not found: {path} Subdomain folder doesn't exist on server Create subdomain in cPanel first, then retry
trigger() HTTP 404 Mvcdeploy controller missing (BigRock 406 misleads check) Auto-fixed: Python FTPs Mvcdeploy.php from localhost and retries. If still fails, check mvc/controllers/ folder exists.
All proxy URLs blocked by mod_security BigRock/HostGator blocks HTTP POST Auto-fixed: FTP fallback is now used for these servers
Cannot connect to dummy server bootstrap_copy.php not uploaded to dummy1 Upload bootstrap_copy.php + files to dummy1.{domain}/ via cPanel File Manager
Could not find or stop Python process Python server was already stopped Normal — just click Start Python Server
mvc.zip not found in webroot mvc.zip wasn't uploaded before trigger() called Click Rocket again — FTP upload and trigger happen together

10. Quick Reference Card

🖥 Python Server

  • Start: click Start Python Server button in UI
  • Stop: click Python Server Running (green button)
  • URL: localhost:8000/config-check to verify configs
  • After any python/main.py or .env change → restart server

📋 New Subdomain Checklist

  • Create subdomain in cPanel
  • Create database + import tables (DB button)
  • Click 🔌 Plug to bootstrap controllers
  • Click ☁ CSS to push CSS
  • Click 🚀 Rocket to deploy MVC code
  • If brand new: also click 📦 Archive for assets

🔧 Why FTP? (vs direct HTTP)

  • HostGator Monarx Security blocks all HTTP POST with large files
  • BigRock mod_security blocks CSS content + PHP file uploads
  • FTP connects directly to the server — bypasses all web security
  • GoDaddy has no mod_security → direct HTTP works fine

📁 Files on Dummy Servers

Upload once to each dummy1.{domain}/:

  • bootstrap_copy.php — copies controllers to subdomains
  • mvcdeploy.php — standalone receiver (GoDaddy style)
  • Cssupdate.php — CSS controller copy
  • css_update_config.php — API key config
  • full_deploy.php — extracts all zips
  • css_sync.php — CSS proxy (fallback)
